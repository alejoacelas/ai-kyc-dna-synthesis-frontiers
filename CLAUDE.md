## Repository Organization

### Data

- `data/customers/full_dataset.csv` — 134 customer profiles (columns: work_url, Type, Name, Institution, Email, Order, customer_info)
- `data/customers/human_baseline_subset.csv` — 40-profile subset used for human baseline comparison
- `data/annotations/ground_truth_flags.json` — manually verified flag determinations for the 40-profile subset
- `data/annotations/blind_gradings.json` — blind re-grading data for LLM-as-judge validation
- `data/annotations/agreements.json` — inter-rater agreement metadata
- `data/annotations/flag_error_comments.json` — human notes on flag accuracy errors
- `data/raw_results/*.json` — raw PromptFoo evaluation outputs (886MB total, Git LFS); four files split by dataset (full vs subset) and prompt (main vs background_work)
- `data/token_pricing.yaml` — per-token pricing for each model

### Processed Data

Generated by `scripts/generate_datasets.py` from raw results:

- `processed/tests.csv` — one row per test assertion (~234k rows). Key columns: `model_label`, `model_type`, `is_human_baseline_dataset`, `test_category`, `metric_name`, `pass`, `institution_country`, `customer_type`
- `processed/responses.csv` — one row per model response (~55k rows). Key columns: `model_label`, `total_cost`, `latency_ms`, `num_web_searches`

### Scripts

**Data pipeline:**
- `scripts/generate_datasets.py` — main pipeline: raw JSON → processed CSVs. Handles latency fetching (OpenRouter API), country classification (Gemini API), source extraction, and pass/fail corrections
- `scripts/merge_blind_gradings.py` — merges blind grading validation with original test data
- `scripts/match_blind_gradings.py` — matches blind gradings to assertion-level results

**Figure and table generation** (all read from `processed/tests.csv` or `processed/responses.csv`):
- `scripts/figures/style.py` — shared colors, model labels, ordering, and matplotlib style setup
- `scripts/figures/compute_paper_metrics.py` — computes Table 1, Table 2, and inline statistics N1–N11
- `scripts/figures/generate_figure2.py` → `paper/figures/figure2_pass_rates_heatmap.png` (Figure 2: pass rates heatmap)
- `scripts/figures/generate_figure2b.py` → `paper/supplementary/figure2b_pass_rates_by_task.png` (supplementary: pass rates by task)
- `scripts/figures/generate_figure2c.py` → `paper/supplementary/figure2c_pass_rates_by_customer_type.png` (supplementary: pass rates by customer type)
- `scripts/figures/generate_figure3.py` → `paper/figures/figure3_cost_breakdown.png` (Figure 3: cost breakdown by model)
- `scripts/figures/generate_figure4.py` → `paper/figures/figure4_cost_vs_performance.png` (Figure 4: cost vs performance scatter)
- `scripts/figures/generate_figure5.py` → `paper/figures/figure5_flag_errors_by_task_region.png` (Figure 5: error breakdown by region and task)
- `scripts/figures/generate_figure_human_vs_ai.py` → `paper/supplementary/figure_human_vs_ai_comparison.png` (supplementary: human vs AI error comparison)
- `scripts/figures/generate_figure_model_rankings.py` → `paper/supplementary/figure_model_rankings.png` (supplementary: model rankings bar chart)
- `scripts/figures/generate_figure_error_by_criterion.py` → `paper/supplementary/figure_error_by_criterion.png` (supplementary: error by criterion)
