## Repository Organization

### Data

- `data/customers/full_dataset.csv` ‚Äî 134 customer profiles (columns: work_url, Type, Name, Institution, Email, Order, customer_info)
- `data/customers/human_baseline_subset.csv` ‚Äî 40-profile subset used for human baseline comparison
- `data/annotations/ground_truth_flags.json` ‚Äî manually verified flag determinations for the 40-profile subset
- `data/annotations/blind_gradings.json` ‚Äî blind re-grading data for LLM-as-judge validation
- `data/annotations/agreements.json` ‚Äî inter-rater agreement metadata
- `data/annotations/flag_error_comments.json` ‚Äî human notes on flag accuracy errors
- `data/raw_results/*.json` ‚Äî raw PromptFoo evaluation outputs (886MB total, Git LFS); four files split by dataset (full vs subset) and prompt (main vs background_work)
- `data/token_pricing.yaml` ‚Äî per-token pricing for each model

### Processed Data

Generated by `scripts/generate_datasets.py` from raw results:

- `processed/tests.csv` ‚Äî one row per test assertion (~234k rows). Key columns: `model_label`, `model_type`, `is_human_baseline_dataset`, `test_category`, `metric_name`, `pass`, `institution_country`, `customer_type`
- `processed/responses.csv` ‚Äî one row per model response (~55k rows). Key columns: `model_label`, `total_cost`, `latency_ms`, `num_web_searches`

### Scripts

**Data pipeline:**
- `scripts/generate_datasets.py` ‚Äî main pipeline: raw JSON ‚Üí processed CSVs. Handles latency fetching (OpenRouter API), country classification (Gemini API), source extraction, and pass/fail corrections
- `scripts/compute_paper_metrics.py` ‚Äî computes numerical claims cited in the paper (Table 1, Table 2, and inline statistics N1‚ÄìN11)
- `scripts/merge_blind_gradings.py` ‚Äî merges blind grading validation with original test data
- `scripts/match_blind_gradings.py` ‚Äî matches blind gradings to assertion-level results

**Figure generation** (all read from `processed/tests.csv` or `processed/responses.csv`):
- `scripts/figures/style.py` ‚Äî shared colors, model labels, ordering, and matplotlib style setup
- `scripts/figures/generate_figure1.py` ‚Üí `paper/figures/figure1_pass_rates_heatmap.png` (pass rates heatmap, 3 model blocks)
- `scripts/figures/generate_figure1b.py` ‚Üí `paper/supplementary/figure1b_pass_rates_by_task.png` (pass rates by task)
- `scripts/figures/generate_figure1c.py` ‚Üí `paper/supplementary/figure1c_pass_rates_by_customer_type.png` (pass rates by customer type)
- `scripts/figures/generate_figure2.py` ‚Üí `paper/figures/figure2_human_vs_ai_comparison.png` (human vs AI error comparison)
- `scripts/figures/generate_figure3.py` ‚Üí `paper/figures/figure3_model_rankings.png` (model rankings bar chart)
- `scripts/figures/generate_figure6.py` ‚Üí `paper/figures/figure6_flag_errors_by_task_region.png` (error breakdown by region and task)
- `scripts/figures/generate_figure7.py` ‚Üí `paper/supplementary/figure7_cost_vs_performance.png` (cost vs performance scatter)
